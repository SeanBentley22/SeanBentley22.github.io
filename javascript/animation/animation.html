Animations

SVG

  • SVG elements are as interactive as normal HTML Elements because every shape we make has an underlying object in the DOM.
  • We can make a basic animation simply by modifying some property of an SVG element via the recursive callback function we pass into requestAnimationFrame().
  • We could alternatively use CSS Animations instead of Javascipt's requestAnimationFrame() in most cases, as well.

Canvas

  • Crucially, shapes drawn to the canvas have no underlying object in the DOM. They are just pixels painted to the screen. There's no way to reference them afterwards. It means that,
    1. We can't use CSS animations (i.e. keyframing properties), and must use requestAnimationFrame()
    2. We probably want to clear the canvas before updating the screen with a new drawing, using a function such as clearRect(). That is, we can't just modify a property like we did when using SVG. All the pixels painted during the last repaint would otherwise remain on the screen.
  • Because of the necessity of clearing everything before you redraw, we likely wouldn't pick Canvas if we had some animation that only moved one small part of a very big drawing. SVG would usually be more suitable in that case.
  • However, because shapes we draw in canvas don't have to go through the extra work of becoming objects in the DOM, with their a data saved and a reference to them available, if we have an animation involving lots of tiny moving shapes, we could be better off using Canvas.
